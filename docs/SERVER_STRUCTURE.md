# Server Structure Blueprint

This document explains the professional structure used by the included demo blueprint. Rename channels, roles and messages to fit your own community.

## Entry Area

Recommended public locked channels:

- rules;
- verification;
- announcements;
- changelog;
- faq.

New members should be able to understand the community before receiving the member role. Official channels should remain locked so staff communication stays clean.

After verification, the bot grants the member role and unlocks internal channels.

## Languages

Language roles are independent from membership roles. This scales better than mixing all languages into one general channel.

The included demo uses:

- Portuguese;
- English;
- Spanish.

Each language role unlocks its own category and voice channels.

## Support

A single public ticket launcher channel should create private ticket channels inside a private ticket category.

Use tickets for:

- account and access issues;
- reports;
- sensitive exploits;
- private support;
- cases that need screenshots or private data.

Use public feedback channels for open bugs, suggestions and ideas.

## Staff

Staff channels should be private and separated from public community discussion.

Recommended staff channels:

- moderation logs;
- staff chat;
- staff checklist;
- ticket archive or transcript references.

## Progression Roles

The demo blueprint includes level roles:

- level 5: Recruit;
- level 10: Warrior;
- level 20: Guardian;
- level 35: Champion.

These are examples. Treat them as a template for your own community progression.

## Economy And Rewards

The demo economy uses coins and CP. Both are spendable resources:

- shop purchases spend coins and CP;
- creature capture attempts spend coins and CP;
- staff can award resources through dashboard controls;
- voice and chat activity can grant resources.

When adapting the blueprint, keep the reward loop simple:

1. Participate.
2. Earn XP and currency.
3. Unlock roles, badges or cosmetic rewards.
4. Spend resources on optional progression.
